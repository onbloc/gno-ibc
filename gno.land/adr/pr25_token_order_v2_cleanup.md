# TokenOrderV2 Union Cleanup

## Context

ZKGM TokenOrderV2 handling exists in both the non-v1 app implementation and the
v1 fork. The cleanup plan in `local_docs/zkgm/33-token-order-v2-cleanup-plan.md`
identified drift from Union's `verify_token_order_v2`,
`protocol_fill_escrow_v2`, `protocol_fill_unescrow_v2`, and
`acknowledge_token_order_v2` paths.

The main risks were accepting invalid UNESCROW sends, minting arbitrary ESCROW
quote denoms, losing protocol fee accounting, and dropping V2 metadata image
association during ack/refund voucher re-mints.

## Decision

Apply the TokenOrderV2 cleanup to both code paths:

- derive wrapped token denoms from ABI-encoded `(path, channel, token[, image])`
  salt input, then keep the Gno-local `ibc/<hex20>` projection
- save token origins with `UpdateChannelPath(path, destinationChannel)` when
  INITIALIZE or lazy ESCROW creation mints a voucher
- require UNESCROW sends to have token origin, reverse-path match, channel
  match, and V1/V2 wrapped-token self-match before burning or accepting coins
- require ESCROW recv quote tokens to match the predicted wrapped denom and
  require `baseAmount >= quoteAmount`
- distribute ESCROW/UNESCROW protocol fees to the relayer after preflighting
  all `uint256 -> int64` conversions
- preserve stored metadata images when ack/refund code re-mints vouchers

## Alternatives Considered

Keeping ESCROW recv permissive and relying on later UNESCROW validation was
rejected because it allows arbitrary voucher identifiers to be minted and makes
TOKEN_ORIGIN state unreliable.

Skipping lazy ESCROW origin saves was rejected because the Gno voucher ledger can
create wrapped tokens outside Union's `can_deploy=true` INITIALIZE path. Without
the compensating origin write, valid later UNESCROW sends would fail the new
self-match guard.

## Consequences

Wrapped voucher denoms generated by Gno change because the salt input is now
Union-compatible. Pre-launch state can regenerate these denoms without a
migration.

Some fixed-proof realcometbls scenarios now assert failure for old packets whose
quote token is no longer the predicted wrapped denom; changing those packet
bytes would require regenerating the proof fixtures.
